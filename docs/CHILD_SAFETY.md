# Child Safety

LearningForge is for child learning practice, so product decisions should minimize personal data, pressure, and open-ended AI exposure.

## Safety Rules

- No open child chat in the MVP.
- No shaming, insults, fear, ranking against peers, or punitive streak language.
- Do not request full name, precise location, school, contact info, photos, or sensitive personal details.
- Parent-entered interests are optional and used only as light presentation motifs.
- Keep child-facing copy short, encouraging, and specific to the current activity.
- AI failures must not expose technical errors to the child.

## Data Rules

Store only what the quest loop needs:

- display name or nickname
- grade level
- optional interests
- preferred theme
- tutor tone
- daily goal
- attempts, rewards, progress, and forge state

Do not store open-ended child chat transcripts because the MVP does not include open chat.

## Review Checklist

- Could a child complete the flow without being asked for unnecessary personal data?
- Does every incorrect-answer message preserve dignity and point to a next action?
- Does every AI-assisted surface have a deterministic fallback?
- Are parent summaries based on real attempts rather than inferred traits?
